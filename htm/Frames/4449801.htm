O Windows Server 2012 Remote Desktop Services é ideal para quem precisa configurar e gerenciar máquinas remotamente.

 

Por que usar os Serviços de Área de Trabalho Remota?

Implantação de aplicativos: você pode implantar rapidamente programas baseados no Windows em dispositivos de computação em toda a empresa. Os Serviços da Área de Trabalho Remota é especialmente útil com programas que são atualizados com frequência, usados raramente ou difíceis de gerenciar.

Consolidação de aplicativos: os programas são instalados e executados em um servidor Host da Sessão da Área de Trabalho Remota, eliminando a necessidade de atualizar os programas em computadores cliente. Isso reduz também a quantidade de largura de banda de rede necessária para acessar os programas.

Acesso remoto: os usuários podem acessar programas que executam em um servidor Host da Sessão da Área de Trabalho Remota em dispositivos como computadores domésticos, quiosques, hardware de baixa potência e sistemas operacionais que não sejam o Windows.

Acesso em filiais: os Serviços da Área de Trabalho Remota proporciona melhor desempenho de programas para usuários em escritórios de filiais que precisam acessar armazenamentos de dados centralizados. Os programas que processam muitos dados às vezes não dispõem de protocolos cliente/servidor otimizados para conexões de baixa velocidade. Programas desse tipo geralmente têm melhor desempenho em uma conexão de Serviços da Área de Trabalho Remota do que em uma rede de longa distância típica.

 

 

Serviços de Área de Trabalho Remota (anteriormente conhecido como Terminal Services) fornece uma funcionalidade semelhante a um, anfitrião centralizado, ou de mainframe, ambiente baseado em terminal em que vários terminais conectar a um computador host. Cada terminal fornece uma canalização para entrada e saída entre um usuário eo computador host. Um usuário pode fazer logon em um terminal, e depois executar aplicações no computador host, acessar arquivos, bases de dados, recursos de rede, e assim por diante. Cada sessão de terminal é independente, com o sistema operacional hospedeiro gestão de conflitos entre vários usuários em disputa por recursos compartilhados.

A principal diferença entre o Remote Desktop Services e do ambiente de mainframe tradicional é que os terminais burros em um ambiente de mainframe só fornecem entrada e saída baseada em caracteres. Um Remote Desktop Connection (RDC) cliente ou emulador fornece uma interface gráfica do usuário completo, incluindo um sistema operacional de desktop Windows e suporte para uma variedade de dispositivos de entrada, como um teclado e mouse.

No ambiente de serviços de desktop remoto, um aplicativo é executado inteiramente no Host da Sessão da Área de Trabalho Remota (Host de Sessão RD) (anteriormente conhecido como um servidor de terminal). O cliente RDC não executa nenhum processamento local de software de aplicação. O servidor transmite a interface gráfica do usuário para o cliente. O cliente transmite a entrada do usuário de volta para o servidor.
